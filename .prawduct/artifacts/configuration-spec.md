# Configuration Specification

<!-- sourced: docs/high-level-design.md § Persistence Model, 2026-02-16 -->

## Configuration Items

### Project-Level Configuration

| Item | File | Purpose | Format | Default |
|------|------|---------|--------|---------|
| Project state | `.prawduct/project-state.yaml` | Master state: classification, decisions, artifacts, build plan | YAML | Copied from `templates/project-state.yaml` |
| Framework location | `.prawduct/framework-path` | Absolute path to prawduct framework directory | Plain text (single line) | Set by `prawduct-init.py` |
| Framework version | `.prawduct/framework-version` | Git SHA of framework at setup/last update | Plain text (single line) | Set by `prawduct-init.py` |
| Bootstrap instructions | `CLAUDE.md` | Entry point for LLM — sources framework CLAUDE.md | Markdown | Generated by `prawduct-init.py` |

### Session-Level Configuration

| Item | File | Purpose | Format | Default |
|------|------|---------|--------|---------|
| Governance state | `.prawduct/.session-governance.json` | Edit tracking, debt tracking, DCP state (including `needs_classification` trigger and `artifacts_verified` list) | JSON | Created by Orchestrator step 4; `needs_classification` set by governance-tracker.sh |
| Activation marker | `.prawduct/.orchestrator-activated` | Timestamp proving Orchestrator was loaded. Cross-repo: must exist at `$CLAUDE_PROJECT_DIR/.prawduct/` (where hooks check); also written to target dir for future direct launches. | ISO-8601 text | Created by Orchestrator step 3 |
| Critic evidence | `.prawduct/.critic-pending` | Timestamp of last framework edit awaiting review | ISO-8601 text | Created by governance-tracker.sh |
| Critic findings | `.prawduct/.critic-findings.json` | Structured Critic review evidence for commit gate | JSON | Created by record-critic-findings.sh |

### Framework-Level Configuration

| Item | File | Purpose | Format | Default |
|------|------|---------|--------|---------|
| Hook registrations | `.claude/settings.json` | Maps Claude Code events to governance hook scripts | JSON | Generated by `prawduct-init.py` (merges with user hooks) |
| Observation schema | `.prawduct/framework-observations/schema.yaml` | Defines valid observation types, statuses, fields | YAML | Maintained in framework repo |
| Detection thresholds | `tools/obs_utils.py` | Tiered pattern detection thresholds | Python constants | meta: 2, build: 3, product: 4 |

## Configuration Mechanism

All configuration is file-based:
- **YAML** for structured project state and observation schema
- **JSON** for session-scoped governance state and Claude Code settings
- **Plain text** for simple values (framework-path, framework-version, activation marker)

No environment variables. No config service. No database. Files are human-readable and git-trackable.

## Configuration Validation

- **Startup validation:** `prawduct-init.py` validates infrastructure state and reports issues as JSON
- **Schema validation:** `scripts/validate-schema.py` checks project-state.yaml against template structure
- **Observation validation:** `capture-observation.sh` validates required fields before writing
- **Missing config behavior:** Hooks fail loudly on missing files. Session Resumption recreates `.prawduct/.session-governance.json` if absent.

## Configuration Changes

- **project-state.yaml:** Modified by skills during the session. Changes are immediate — no restart required.
- **.claude/settings.json:** Changes take effect on next tool invocation (hooks re-read on each call).
- **Framework files (skills, templates):** Read from disk on demand. Changes take effect immediately when the file is next read.

## Secrets Management

Not applicable — Prawduct requires no API keys, no tokens, no credentials. All configuration is non-sensitive text files.
